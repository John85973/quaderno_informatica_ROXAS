ARGOMENTI AFFRONTATI SUL DATABASE:


DATABASE

Un database e una raccolta di dati organizzata in tabelle, ognuna rappresentante un'entita (es. professori, studenti).

Le tabelle sono composte da righe (record) e colonne (attributi). Le righe rappresentano i record e le colonne

gli attributi di ogni entita.


ENTITA

Le entita sono oggetti reali (es. una persona, una macchina) e possiedono attributi (es. nome, data di nascita).

Le entita devono avere attributi che descrivono le loro caratteristiche.


CHIAVE PRIMARIA

La chiave primaria identifica univocamente un'entita, come ad esempio il codice fiscale di una persona,

che e un attributo unico che non si ripete. Per indicare un attributo come chiave primaria si scrive "PK".

La sua sintassi nel linguaggio SQL viene scritto ad esempio:


CREATE TABLE Studenti (

ID_Studente INT PRIMARY KEY,

Nome VARCHAR(100),

Eta INT

);


CHIAVE ESTERNA

La chiave esterna, o foreign key, e un attributo che viene utilizzato per creare un collegamento

con un'altra entita, facendo riferimento alla chiave primaria di quest'ultima.


Esempio:

CREATE TABLE Studenti (

ID_Studente INT PRIMARY KEY,

Nome VARCHAR(100),

Eta INT

);


CREATE TABLE Corsi (

ID_Corso INT PRIMARY KEY,

Nome_Corso VARCHAR(100),

ID_Studente INT,

FOREIGN KEY (ID_Studente) REFERENCES Studenti(ID_Studente)

);


CARDINALITA

Definisce la relazione tra le entita, che puo essere di quattro tipi:

1:1: un'entita e collegata a una sola altra.

1:N: un'entita e collegata a molte altre entita.

N:1: molte entita sono collegate a una sola.

N:N: molte entita sono collegate a molte altre.


MODELLO E/R

Sarebbe un modello concettuale che viene utilizzata per rappresentare la struttura logica di

un database. Il suo scopo e di descrivere attraverso una rappresentazione grafica le entita e

i legami tra di loro. Sono composti da:

  • Entita
  • Attributi
  • Relazione
  • Cardinalita
  • Chiave primaria
  • Chiave esterna



QUERY

Le query sono delle istruzioni che permettono di interagire con il database per eseguire operazioni come lettura, inserimento e aggiornamento. Ecco un esempio di query:

SELECT cognome, eta

FROM Studenti

WHERE eta < 18;


COMANDI BASE (SQL)

CREATE DATABASE: crea un database.

DROP DATABASE: elimina un database.

CREATE TABLE: crea una tabella.

DROP TABLE: elimina una tabella.


ALTER

Il comando ALTER viene utilizzato per modificare una tabella. Puo essere usato per aggiungere, eliminare o cambiare colonne all'interno di una tabella.

ADD: aggiunge una colonna.

DROP: elimina una colonna.

CHANGE: modifica una colonna esistente.


INSERT INTO

Il comando INSERT INTO viene usato per inserire nuovi dati in una tabella. Esempio:

INSERT INTO nometabella

VALUES ("Marco", "Rossi", "JDW93H284H");


SELECT

Il comando SELECT permette di estrarre dati da una tabella. Esempio:

SELECT cognome, eta

FROM Studenti

WHERE eta < 18;
